# Glossary of Terms

Quick reference for technical terms used in this lesson and throughout the course.

---

## A

**API (Application Programming Interface)**
> A way for different applications to communicate with each other programmatically.

**Automation**
> Making tasks happen automatically without human intervention.

---

## C

**Connection**
> A link between two nodes that defines how data flows in a workflow.

**Credentials**
> Authentication information (usernames, passwords, API keys) needed to connect to services.

**Cron Job**
> A scheduled task that runs at specific times/intervals.

---

## D

**Docker**
> A platform for running applications in isolated containers.

**Docker Compose**
> A tool for defining and running multi-container Docker applications.

---

## E

**Execution**
> A single run of a workflow from start to finish.

**Execution History**
> A log of all past workflow runs, including success/failure data.

---

## H

**Horizontal Scaling**
> Adding more worker instances to handle increased load.

---

## J

**JSON (JavaScript Object Notation)**
> A standard format for structuring data that n8n uses internally.

---

## N

**Node**
> A single block/step in a workflow that performs one specific action.

**n8n**
> Pronounced "n-eight-n" or "nodemation" - the workflow automation platform.

---

## O

**OCR (Optical Character Recognition)**
> Technology that extracts text from images or PDFs.

**Open-source**
> Software with source code that anyone can inspect, modify, and distribute.

---

## P

**PostgreSQL**
> An open-source relational database that n8n uses to store workflows and data.

**Polling**
> Regularly checking a service for new data (e.g., checking email every 5 minutes).

---

## Q

**Queue**
> A line of jobs waiting to be processed by workers.

**Queue Mode**
> n8n's production mode where workflows are queued in Redis and processed by workers.

---

## R

**Redis**
> An in-memory data store used by n8n for job queuing and caching.

**REST API**
> A standard way to build web APIs using HTTP requests.

**ROI (Return on Investment)**
> The value/benefit gained compared to the cost/effort invested.

---

## S

**Self-hosted**
> Running software on your own servers instead of using a cloud service.

**SQLite**
> A lightweight database suitable for development but not production.

**SSL/TLS (Secure Sockets Layer / Transport Layer Security)**
> Encryption protocols for secure HTTPS connections.

---

## T

**Trigger**
> The event or condition that starts a workflow execution.

**Trigger Node**
> The first node in a workflow that defines when it should run.

---

## V

**VPS (Virtual Private Server)**
> A virtual machine hosted in the cloud that you can use to run n8n.

---

## W

**Webhook**
> A way for external services to trigger n8n workflows by sending HTTP requests.

**Worker**
> A process that executes workflows from the queue.

**Workflow**
> A sequence of automated actions (nodes) that accomplish a task.

---

| Acronym | Stands For | Meaning |
|---------|------------|---------|
| **API** | Application Programming Interface | How apps talk to each other |
| **CI/CD** | Continuous Integration/Deployment | Automated testing and deployment |
| **CRM** | Customer Relationship Management | System for managing customers |
| **CRUD** | Create, Read, Update, Delete | Basic database operations |
| **HTTPS** | HTTP Secure | Encrypted web traffic |
| **JSON** | JavaScript Object Notation | Data format |
| **OCR** | Optical Character Recognition | Text extraction from images |
| **REST** | Representational State Transfer | API architecture style |
| **ROI** | Return on Investment | Value of automation |
| **SSL** | Secure Sockets Layer | Encryption protocol |
| **UI** | User Interface | Visual interface |
| **VPS** | Virtual Private Server | Cloud server |
